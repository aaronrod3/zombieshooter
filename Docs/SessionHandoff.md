## Last completed (2026-07-21, same-day follow-up #6)

Dev recompiled and walked toward the (invisible, per known gap) test zombie - **no reaction at all**. Diagnosed via `EditorToolset.LogsToolset.GetLogEntries` + `BlueprintTools.compile_blueprint`, not guessing - and it's a bigger finding than the missing-`BehaviorTree`-assignment fix from last round (that fix is confirmed saved to disk now, `DA_ZS_ZombieConfig_Shambler.uasset`/`AZombieCharacter.uasset` both show modified in git).

**Root cause: a second, broader instance of this session's Live Coding corruption bug.** All three custom BT task Blueprints that cast to `AZombieAIController` - `BTTask_MeleeAttack`, `BTTask_StartInvestigationTimer`, `BTTask_StartIdleDwell` - have a broken `Cast To ZombieAIController` node (`this cast has an invalid target type - was the class deleted without a redirect?`), confirmed via `BlueprintTools.compile_blueprint` failing on all three with matching errors (`Could not find a function named "TriggerMeleeAttack"/"StartInvestigationTimer"/"StartIdleDwell"`). **This is NOT limited to Blueprints whose own parent class changed** (the earlier `BP_ZS_PlayerCharacter` incident) - `AZombieAIController`'s header wasn't touched this round at all. The real mechanism: Live Coding reinstances every class in the module on every patch, so any Blueprint anywhere with a cast to a native class in that module is at risk, surfacing only once the engine actually loads that Blueprint (here, triggered by an `unreal-mcp` property read last round, not by the patch itself). Generalized as a `CLAUDE.md` lesson - this is a project-wide risk going forward, not a one-off.

**Fix is manual, not something `compile_blueprint` alone resolves** - confirmed by trying it: the broken cast node has to be deleted and a fresh `Cast To ZombieAIController` recreated in each of the 3 assets, then recompiled and saved. Dev chose to do this by hand (offered an `unreal-mcp` node-editing attempt, dev preferred manual given this is real ShooterGame-derived content worth not risking). **In progress as of this handoff - not yet confirmed fixed.**
