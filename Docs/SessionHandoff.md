**Found and diagnosed a real bug the same round (2026-07-21, PIE - no player mesh visible):** dev reported PIE spawning at a PlayerStart with full movement/camera but no visible character mesh. Root cause, found via `EditorToolset.LogsToolset.GetLogEntries` (not guessed): the dev's recompile went through **Live Coding**, not a full rebuild, and Live Coding's class re-instancing broke `BP_ZS_PlayerCharacter`'s parent-class link to native `AZSPlayerCharacter` (log: `BlueprintGeneratedClass BP_ZS_PlayerCharacter_C is not a child class of Class ZSPlayerCharacter`) - a known Live Coding limitation when new `UPROPERTY`s get added to a class with existing Blueprint children (this round added several). That broke `AZSGameMode`'s `ConstructorHelpers::FClassFinder` lookup for the Blueprint, which silently fell back to spawning the **raw C++ `AZSPlayerCharacter` class** instead of the Blueprint - no crash, no visible error. Since the character's mesh is entirely weapon-config-driven (`RefreshBodyMeshFromWeapon`, fed by `StartingWeaponConfig`, which only lives on the *Blueprint's* CDO, not the raw C++ class), no weapon ever equipped and no mesh was ever assigned - exactly the reported symptom, while movement/camera (pure C++, no BP needed) worked fine. **Confirmed self-healed**: re-checked `BP_ZS_PlayerCharacter`'s parent class via `BlueprintTools.get_parent` just now - correctly resolves to `ZSPlayerCharacter` again (fixed itself when the Blueprint got reopened/recompiled). Everything else checked out fine along the way and was **not** the cause: `AZSGameMode`→BP wiring, `StartingWeaponConfig`→`DA_ZS_WeaponConfig_AssaultRifle`→`TP_Mesh` (valid Manny mesh ref), mesh visibility flags (`bHiddenInGame`/`bVisible`/`bOwnerNoSee` all normal), World Settings GameMode override (`None`, correctly inherits the project's `ZSGameMode` default). The two `PlayerStart` actors in `Lvl_ThirdPerson` (hence "Player Start 2") are also a non-issue - just two spawn points in the map. Logged as a new `CLAUDE.md` lesson: check the Output Log for `is not a child class of` before anything else when something breaks right after a Live Coding recompile with header changes, and prefer a full editor Compile or `Build.bat` over Ctrl+Alt+F11 when new `UPROPERTY`/`UFUNCTION`s were added to a class with Blueprint children.
