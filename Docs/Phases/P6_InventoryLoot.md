## Known gaps / not resolved this session
- **P5's hotbar doesn't reference this component yet.** `AZSPlayerCharacter::HotbarSlots` still holds direct `UZSWeaponConfig*` references (a placeholder pattern, unchanged from before this phase existed) rather than referencing stacks held in `UZSInventoryComponent::CarrySlots`. Wiring hotbar slots to actual carried/equipped weapon items (so you can only hotbar a gun you're actually carrying) is real follow-up work, not done here.
- **No pickup→hotbar path.** Picking up a weapon via `AZSWorldItemActor` adds a `UZSWeaponConfig` reference to `CarrySlots` like any other item, but nothing moves it into a `HotbarSlots` entry automatically - there's no "equip from inventory" UI/action yet to do that by hand either.
- **Container "already looted" edge case**: a container whose `LootTable` rolls zero items (bad luck, or an unset `LootTable`) starts as non-interactable (`bIsInteractable` follows `ContainerSlots.Num() > 0`), which reads slightly oddly as "already looted" for something that was never interacted with. Not worth extra complexity for this session; revisit if it's confusing in practice.
- **Weight/capacity/rarity-pool numbers are all placeholder defaults** (`BaseCarryWeight = 25`, `OverloadWeightRatio = 1.5`, etc.) - not tuned against any playtest data, same caveat as P5's ammo-scarcity gap.
